# Alisnart Page Pattern Library

Use these structures when creating or redesigning pages.

## 1. Homepage Pattern

Goal: convince a client within 10 seconds that Alisnart can design and deliver real light-art/night-tourism projects.

Structure:

1. Fixed header with visible navigation and language switch.
2. Full-bleed or wide hero using real project/product visual.
3. Claim: "Use light to rebuild scenic-area traffic memory" or equivalent market-specific claim.
4. Primary CTA: View cases / View products.
5. Secondary CTA: Contact / Quote.
6. Proof metrics: years, cases, served regions.
7. Service categories: scenic night tourism, commercial street, ancient town, hotel/resort, city renewal, performance/event.
8. Featured cases/products with real images.
9. Why it operates: flow, photos, stay time, maintenance, content renewal.
10. Contact block.

Homepage rule: do not show internal test posts or AI-only GEO pages as primary homepage content.

## 2. Case Page Pattern

Goal: make a client understand what was built, why it mattered, and whether it can be adapted to their site.

Structure:

1. Hero image with project name and scenario.
2. Quick facts:
   - Scenario
   - Product type
   - Materials
   - Control system
   - Installation environment
   - Operation objective
3. Problem / client objective.
4. Design solution.
5. Visitor journey or night-tour route.
6. Engineering details:
   - Structure
   - Waterproofing
   - Control
   - Maintenance
   - Power/safety
7. Result / operational value.
8. Related products or cases.
9. CTA.

Case page rule: every case must include at least one image and one engineering/detail paragraph.

## 3. Product Page Pattern

Goal: turn one lighting-art concept into a reusable sales/proposal module.

Structure:

1. Product hero image.
2. Product name and one-sentence fit:
   - "A luminous ring photo-point installation for scenic entrances and main visitor-route nodes."
3. Use scenarios:
   - Scenic area entrance
   - Lakeside
   - Commercial street
   - Ancient town
   - Hotel entrance
4. Visitor value:
   - Check-in/photo
   - Interaction
   - Traffic guidance
   - Performance/route memory
5. Technical spec:
   - Size range
   - Materials
   - LED/control
   - IP rating
   - Maintenance access
6. Operation model:
   - Normal night mode
   - Festival mode
   - Interactive mode
7. Proposal assets:
   - Website image 16:9
   - Product image 4:5
   - Social cover 1:1
8. CTA.

Product page rule: prompt text must not be displayed to users. Generated images and product explanation are shown; prompts stay internal.

## 4. Insight / GEO Page Pattern

Goal: answer one buyer or AI-search question better than forum posts and generic competitors.

Structure:

1. Question as H1.
2. Direct answer in the first paragraph.
3. 3-5 verifiable parameters:
   - IP rating
   - control protocol
   - material
   - install cycle
   - budget range
4. Explanation by scenario.
5. Common mistakes.
6. What information is needed for a quote.
7. Related product/case links.
8. Contact.

GEO rule: write for humans first. The answer should be quotable without looking like hidden machine text.

## 5. Light Designer / Quote Tool Pattern

Goal: make the tool feel like a practical project estimator, not a low-effort form.

Structure:

1. Scenario selector with visual icons or clear segmented buttons.
2. Inputs:
   - project type
   - route length / area
   - budget
   - objective
   - timeline
3. Output:
   - concept route
   - core installation type
   - rough budget logic
   - required materials from client
   - next step CTA
4. Optional:
   - generated proposal outline
   - export/share text

Tool rule: the first result must be useful even before AI integration.
